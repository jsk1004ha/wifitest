# 버전 노트

## 2026-03-30

- 기존 `tcp.py`를 구조화된 네트워크 진단 실행기로 교체했습니다.
- `tcp`, `tls`, `udp_dns`, `udp_stun`에 대한 명시적 진단 타입을 추가했습니다.
- 대상별 카테고리를 포함하는 JSON 기반 대상 카탈로그(`targets.json`)를 추가했습니다.
- GitHub Pages 경로처럼 정확한 배포 페이지 검사를 위한 선택적 HTTP URL 진단을 추가했습니다.
- CSV 내보내기, 네트워크 라벨, 진단 타임스탬프, 해석/검사된 IP 메타데이터를 추가했습니다.
- TCP/TLS/UDP 진단이 실패를 보고하기 전에 해석된 각 IPv4 주소를 모두 재시도하도록 다중 IP 처리 방식을 개선했습니다.
- 기본 대상 세트에 Riot, VALORANT, Steam, Epic, Blizzard, Battle.net, Nexon 관련 일반 게임 및 런처 웹사이트를 확장 추가했습니다.
- 기본 Windows 터미널 인코딩에서도 스크립트가 동작하도록 이모지 출력을 제거했습니다.
- UDP 호스트명 해석과 응답 검증을 수정했습니다.
- 타임아웃 제어, 단일 대상 검사, JSON 내보내기를 위한 CLI 플래그를 추가했습니다.
- `UDP.py`를 더 명확한 결과 상태를 가진 UDP DNS 전용 진단 도구로 재구성했습니다.
- 설정, 사용법, 결과 해석 가이드를 담은 `README.md`를 추가했습니다.
- 현재 프로그램 설계, 범위 제한, 구현 단계를 담은 `agent.md`를 추가했습니다.
- 엔진 시작 실패가 조용한 실행 실패처럼 보이지 않도록, WPF 컨트롤러에서 stderr와 종료 코드를 통해 오류를 명확히 표시하도록 개선했습니다.
- 누락된 엔진 바이너리, 시작 중, 실행 중, 중지/강제 중지 경로에 대해 UI 상태 전이를 더 명확하게 추가했습니다.

## 범위

- 우회, 스푸핑, 패킷 단편화, 방화벽 회피 로직은 포함하지 않습니다.

# Implementation Plan: Advanced Evasion Fixes (DPI By-pass)

## 1. 기존 기법(OOO 및 파편화) 실패 원인 분석
테스트 결과(STUN TIMEOUT + 게임 사이트 TIMEOUT)를 확인했습니다. 프로세스를 `python.exe`로 인식하여 엔진이 정상적으로 개입(Intercept)했음에도 타임아웃이 났다는 것은, 현재 방화벽(학교 등)이 다음 중 하나의 강력한 대응 시스템을 갖추고 있음을 뜻합니다.

1. **Out-of-Order (OOO) 패킷 즉시 차단:** 비순차적으로 패킷이 들어올 경우 재조립하지 않고 변조 시도로 간주하여 패킷 덩어리 전체를 Drop(폐기)하는 깐깐한 룰이 적용됨.
2. **Buffer Reassembly:** 1바이트나 5바이트로 아주 작게 쪼개 보내 순차적으로 들어오더라도, DPI가 곧바로 처리하지 않고 버퍼에 대기시켰다가 나머지 페이로드가 도착하면 하나의 완전한 SNI(Server Name Indication) 문자열을 조립해 버린 후 차단 도메인이면 닫아버림.

이러한 **적극적 재조립 방화벽(Stateful DPI)**은 단순히 패킷을 작게 쪼개는 것만으로는 뚫을 수 없습니다. DPI의 '상태 추적기(State Machine)' 자체를 고장내거나 속여야 합니다.

## 2. 해결책 제안 (개선점 적용 계획)

단순 파편화를 넘어, DPI 자체를 교란하는 가장 효과적이고 확실한 기술 2가지를 `Diverter.cpp`에 결합하여 구현합니다.

### 기법 A: TTL (Time-To-Live) 기반 가짜 세션 주입 (TTL Spoofing)
방화벽은 사용자와 물리적으로 가까운 위치(1~3 홉 이내)에 있습니다. 이를 역이용합니다.
* **로직:** 진짜 `ClientHello` 패킷을 전송하기 직전에, 내용 전체가 **의미 없는 쓰레기 데이터(Garbage) 또는 가짜 도메인**으로 채워진 TCP 페이로드를 조작해 보냅니다. 단, 이 가짜 패킷의 IP 헤더의 **TTL(수명) 값을 `2` 또는 `3`**으로 아주 낮게 설정합니다.
* **효과:** 방화벽(DPI)은 이 가짜 패킷을 수신하고 "아, 이 연결은 그냥 잡다한 연결이구나" 하고 세션을 허용 처리합니다. 그러나 이 가짜 패킷은 백본망 라우터에 도달하자마자 수명(TTL)이 다해 폐기되므로 **진짜 도착지 서버(롤, 발로란트)에는 절대 도착하지 않습니다.**
* 그 후, 원래의 진짜 `ClientHello` 패킷을 정상적인 TTL 값으로 전송합니다. 방화벽은 이미 이 세션을 '안전함'으로 판단했기 때문에 아무 검사 없이 프리패스(Bypass)시킵니다.

### 기법 B: Window Size (TCP 윈도우) 변조 결합
파편화의 효과를 강제로 살리기 위해, TCP 헤더의 `Window Size` 옵션을 조작해 방화벽이 일시적으로 버퍼 윈도우 크기를 착각하게 만들면 Reassembly(재조립) 공격 방어막을 교란할 수 있습니다.
